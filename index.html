编辑器之神——vim

发表于 2020-09-08
本文字数： 1.7k 阅读时长 ≈ 2 分钟

vi ：
打开文件
vi filename(s) [n] #打开(多个)文件[到第n行]

编辑模式：
    ctrl k：确认输入模式
    ctrl s：锁定，记忆输入
    ctrl q：解锁，显示输出
    ctrl o: 暂退编辑模式
    ctrl g j/k: 相当于↓,↑
    ctrl j,m：相当于enter
    ctrl h：相当于backspace
    ctrl i: 相当于tab
    ctrl t: 向右缩进
    ctrl n,p：向下，上 选择粘贴一行
    ctrl y：对应粘贴上一行
    ctrl w/u: 向前删
    ctrl e: 撤销
    ctrl rn: 后退n步

    ctrl x：
    ctrl a：
    alt q：标记

命令模式：
    a: 从光标所在的下一个字符处开始插入
    A: 从光标所在行的最后一个字符处开始插入
    i: 从光标所在处插入
    I: 在所在行的第一个非空格符处开始插入
    o: 在光标所在的下一行插入新的一行
    O: 在光标所在的上一行插入新的一行
    r: 取代一个字符
    R: 一直取代，直到按下ESC为止
    s: 剪切当前字符开始插入
    S: 剪切当前行开始插入
    C: 剪切到行尾开始插入
    cc: 剪切当前行开始插入
    ctrl c: 剪切光标后开始插入
    ncj: 如向下删除n行开始插入
    cw：更改光标所在处的字
    cnw：更改n个字
    ZZ: 保存退出

    v: 字符选择
    V: 行选择
    Ctrl v: 块选择
    y: 将反白的地方复制起来
    d: 将反白的地方删除掉

    yw: 复制一个单词
    [n]yy: [n]行复制
    [n]x: 向后剪切一[n]个字符，(cut)
    [n]X: 向前剪切一[n]个字符
    p: 粘贴到光标下一行
    P: 粘贴在游标上一行
    dw: 删除一个单词(delets)
    D: 删到行尾
    [n]dd: 删[n]行
    dG: 删到文尾
    d[n]gg: 删到文首,删n行
    ctrl g：显示行数百分比
    ctrl r: 重做上一个动作(redo)
    u: 复原前一个动作(undo)
    ctrl z: 撤销
    .: 重复操作

    /, ?word：向光标之上，下查找word
    n: 重复正向查找操作
    N: 重复反向查找操作

    [n]h, ← 左移n
    [n]j, ↓, + 下移n
    [n]k, ↑, - 上移n
    [n]l, → 右移n
    [n] space: 右移n
    [n] enter: 下移n
    b: 上一个单词头
    w: 下一个单词头
    e: 下一个单词尾
    ff: 光标找到本行下一个f
    ^, 0, home 行首
    $, end 行尾
    H: 光标移到屏幕最上方那一行的第一个字符
    M: 光标移到屏幕中央那一行的第一个字符
    L: 光标移到屏幕最下方那一行的第一个字符
    ctrl u: 上移半页
    ctrl d: 下移半页
    ctrl f: 相当于PgUp
    ctrl b: 相当于PgDn
    ctrl p: 下一行
    ctrl n,m: 下一行
    [n]gg: 首行，第n行
    G: 末行
    ctrl t: 
    J: 将光标所在列不下一列的数据结合成同一列
    缩徘: <<,>>
    段数切换：{/}
    括号匹配及切换：%

末行模式：
    :q! #强制退出
    :[n1,n2]w [f_n] #[n1到n2行]保存[到f_n]
    :wq! , :x , ZZ #三者相同，保存退出
    :[v]ps 文件路径 #分屏; ctrl w j/k 换页
    :n [f_n] #编辑下一个档案, [新建f_n]
    :N #编辑上一个档案
    :files #列出目前这个vim开启的所有档案
    :set #显示用户改更改的设定参数
    :set [no]nu #[取消]设定行号
    :set all #显示目前所有的环境参数设定值
    :set nohlsearch #设定是否将搜寻的字符串凸显
    :set [no]autoindent #是否自动缩排？autoindent 就是自动缩排
    :syntax on/off #相关语法显示颜色
    :n1,n2s/o/n/g[c]  #n1到n2行[:1,$s 全局]的o变n；c确定提示
    :e . #执行命令
    :! command #暂时离开vi执行command

6.换行符
cat -A filename #查看
DOS: ^M$ 有CR与LF两个符号
Linux: $ 仅有 LF
相互间转换
dos2unix [-kn] file [newfile]
unix2dos [-kn] file [newfile]
-k ：保留该档案原本的mtime时间格式
-n ：保留原本的旧档，将转换后的内容输出到新档案
7.编码
iconv –list #查看支持的语系数据
iconv -f 原编码 -t 新编码 filename [-o newfile] #转换
-f ：from
-t ：to
-o file：如果要保留原本的档案

可用电脑一台，8G左右u盘一个 (^干净的)
[^干净的]:没被玷污的！！！

*一.操作系统下载
官方网站选择不同版本的镜像

*二.下载启动盘制作工具

*三.制作启动盘
1.打开UltraISO
2.继续试用
3.文件->打开->选择镜像文件->打开->启动->写入硬盘映象->写入
注意：硬盘驱动器选择u盘

*四.安装操作系统
1.插上u盘
2.进入BIOS（不同电脑不一样，自己百度，华硕主板F2）
3.选择启动项，把u盘拖到第一个
4.保存退出（F7）
5.接下来就是鼠标操作，识字就行

小白到此结束（操作系统自动安装驱动程序）!!!

2独显的驱动程序

3其他设备驱动程序

build_mongodb

发表于 2020-08-31 更新于 2020-09-01
本文字数： 443 阅读时长 ≈ 1 分钟

安装mongodb
sudo apt(yum) update #更新源
sudo apt(yum) upgrade #更新包
sudo apt install mongodb #下载mongodb
失败E: 软件包 mongodb-server 没有可安装候选
注释旧源，添加新源
vim /etc/apt/sources.list
deb [by-hash=force] https://mirrors.tuna.tsinghua.edu.cn/deepin panda main contrib non-free
udo apt(yum) update #更新源
sudo apt(yum) upgrade #更新包
mongod #打开mongodb shell(说明成功)

查看状态
systemctl start mongodb #启动
systemctl status mongodb #查看
systemctl restart mongodb #重启

卸载
sudo systemctl stop mongodb
sudo apt purge mongodb
sudo apt autoremove

基于hexo搭建个人博客

发表于 2020-08-31
本文字数： 2.3k 阅读时长 ≈ 2 分钟

#deepin v20:
1.安装Git:
sudo apt-get install git
git –version

2.NVM安装 Node.js 和 npm:
下载并安装nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.35.3/install.sh | bash
nvm list-remote #获取一系列nodejs版本
nvm install node #安装node #node -v #npm -v
npm install -g cnpm -registry=https://registry.npm.taobao.org

3.安装hexo
cnpm install -g hexo-cli #hexo -v
npm ERR! Error: EACCES: permission denied, access ‘/usr/local/lib/node_modules问题
1.sudo chmod 775 /usr/local/lib/node_modules
2.sudo chown -R $USER /usr/local/lib/node_modules #推荐，改变目录拥有者
apt –purge remove hexo #卸载hexo

4.初始化hexo
hexo init myblog
cd myblog //进入这个myblog文件夹
cnpm install

5.0
注册coding，gitee，github等用户

5-1.coding托管(国内的平台，网站是中文)
创建项目
选择devops
创建代码仓库
选git，启用启用README.cd文件初始化项目
su
git config –global user.name “your_git_name”
git config –global user.email “your_git_email”
git config user.name #检查
git config user.email #检查
ssh-keygen -t rsa -C “your_git_email” #创建SSH
cat ~/.ssh/id_rsa.pub 添加到 github的ssh key #放置公钥
sudo vi .ssh/config
Host git.coding.com
User your_email
Hostname ssh.git.coding.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa
Port 443
ssh -T git@git.coding.com #查看是否成功
vi _config.yml #配置文件
/deploy:
type:git
repo:仓库地址
branch:master
cnpm install –save hexo-deployer-git #安装上传插件
hexo g -d
静态网页，立即发布，(复制地址)

5-2.GitHub托管(国外网站，是英文)
创建一个仓库
仓库名：用户名.github.io(别为为什么)
选public，勾README
su
git config –global user.name “your_git_name”
git config –global user.email “your_git_email”
git config user.name #检查
git config user.email #检查
ssh-keygen -t rsa -C “your_git_email” #创建SSH
cat ~/.ssh/id_rsa.pub 添加到 github的ssh key #放置公钥
ssh -T git@github.com #查看是否成功
失败ssh: connect to host github.com port 22: Connection refused
方法一，添加以下内容
sudo vi .ssh/config
Host github.com
User 12345@qq.com
Hostname ssh.github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa
Port 443
方法二，加一步ssh-add
cd ~
ssh-add
ssh -T git@github.com #查看是否成功
vi _config.yml #配置文件
/deploy:
type:git
repo:仓库地址(点击Use SSH)
branch:master
cnpm install –save hexo-deployer-git #安装上传插件
hexo g -d
打开项目，点Settings，找到Github pages，选择maser branch
得到你的网址

6.发布文章
hexo n “我的第一篇文章” #创建
vim /home/../soure/_posts/我的第一篇文章.md #编辑
cd /home/../myblog
hexo clean 清理
hexo g 生成
hexo d 部署
hexo s 启动
